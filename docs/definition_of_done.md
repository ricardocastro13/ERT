# Definition of Done (DoD) — Lab 7

## DoD — Requirement (REQ-###)
Um requisito é considerado "Done" quando:

1. Tem ID e descrição clara
2. Está classificado (FR/NFR)
3. Não é ambíguo
4. Não define implementação (sem "como")
5. Tem 2–4 acceptance criteria
6. Está ligado a use cases
7. Impacto da variante está identificado
8. Tem forma de validação (teste/demo)

---

## DoD — User Story (US-###)
Uma user story é considerada "Done" quando:

1. Tem ator e valor definido
2. Tem acceptance criteria claros
3. Cumpre todos os acceptance criteria
4. Foi testada
5. Não tem erros críticos
6. Está documentada
7. Foi aceite pelos stakeholders
